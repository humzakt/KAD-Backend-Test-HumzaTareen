§4.1 HTTP Status Code Matrix (400 / 401 / 404 / 409 / 422)

400: Malformed JSON, bad timestamps (Pydantic errors remapped), missing/too-long Idempotency-Key.
401: Missing or invalid Bearer token.
404: Job not found or cross-user access.
409: Overlapping time window on same asset.
422: Business rule violation (unknown asset, past start_time, bad duration, etc.).

service/api/error_handlers.py — remaps Pydantic RequestValidationError from 422 to 400 (line 19)
service/api/dependencies.py — get_current_user() raises 401 (line 29)
service/api/routes.py — 404 on missing job (line 40), 400 on missing key (line 55)
service/services/validation_service.py — returns 422 for each business rule (lines 41-63)
service/services/job_orchestrator.py — returns 409 on overlap (line 83)
service/constants.py — all status codes are named constants (lines 13-19)

§4.5 Atomic Overlap Check

§4.5 Half-open interval overlap: [start, end)

SQL: WHERE start_time < ? AND end_time > ? (comparing against proposed end and start).

BEGIN IMMEDIATE acquires SQLite's write lock at transaction start. Concurrent callers block on busy_timeout, then see the committed state.

Code Walkthrough

service/constants.py

Centralizes every magic string and number in the application. No other module hardcodes HTTP status codes, MQTT topic names, validation limits, Socket.IO event names, or error message templates. This makes the codebase greppable, refactor-safe, and easy to audit.

service/config.py

Loads all configuration from environment variables with sensible defaults. Parses seed.json at import time into three lookup structures: USERS_BY_TOKEN, USERS_BY_ID, VALID_ASSETS.

service/models.py

Defines the domain model layer:

  • JobState — enum backed by string constants from constants.py
  • VALID_TRANSITIONS — dict mapping each non-terminal state to its allowed successors
  • is_valid_transition(current, proposed) — guard function used by MQTTBrokerService
  • JobCreate — Pydantic model for inbound POST body
  • JobResponse — Pydantic model for outbound job representation

service/providers/ — Protocol Definitions

Four Python Protocol classes define the contracts between layers. This is the Dependency Inversion principle: high-level modules (orchestrator, routes) depend on abstractions, not concrete implementations.

ProtocolFileKey MethodsImplementor
AuthProvider providers/auth.py authenticate(token) → dict|None TokenAuthService
JobRepository providers/repository.py initialize(), find_by_id(), insert_if_no_overlap(), update_state(), + 4 more SQLiteJobRepository
MessageBroker providers/broker.py start(on_result), stop(), publish_dispatch(job) MQTTBrokerService
RealtimeNotifier providers/notifier.py notify_job_update(user_id, payload) SocketIONotifierService

service/services/ — Implementations

ServiceFileResponsibility
TokenAuthService services/auth_service.py Resolves static Bearer tokens from seed data to user dicts.
SQLiteJobRepository services/job_repository.py SQLite persistence with WAL mode. Thread-safe via threading.local() (per-thread connections). BEGIN IMMEDIATE for atomic overlap detection. All SQL as module-level constant strings.
JobValidationService services/validation_service.py Stateless validator enforcing all 6 business rules from §4.4. Returns (status_code, message) tuples on failure.
MQTTBrokerService services/broker_service.py paho-mqtt client with loop_start() background thread. Publishes 5-field dispatch. Subscribes to results and validates state machine transitions before persisting.
SocketIONotifierService services/notifier_service.py Socket.IO AsyncServer wrapper. Token auth from query string. Per-user rooms (user:<id>). Emits job_update events scoped to job owner's room.
JobOrchestrator services/job_orchestrator.py Coordinates the full job creation lifecycle: idempotency check → validation → atomic overlap check + insert → MQTT dispatch. Single entry point for all job mutation logic.

service/api/ — API Layer

FileResponsibility
api/routes.py Thin FastAPI route handlers. Delegates to JobOrchestrator. Handles HTTP-level concerns only: auth, Idempotency-Key validation, status codes.
api/dependencies.py FastAPI Depends() functions. get_current_user() extracts Bearer token and resolves via AuthProvider. get_registry() returns the global registry.
api/error_handlers.py Remaps Pydantic RequestValidationError (which FastAPI defaults to 422) to 400 Bad Request. This is the 400-vs-422 split required by the spec.

macOS-Specific Notes

Mosquitto Docker Issue on macOS

Docker Desktop for macOS runs containers inside a lightweight Linux VM. This introduces a networking indirection layer that can cause Mosquitto to be unreachable from the host at localhost:1883.

Symptoms

  • ConnectionRefusedError when the service tries to connect to MQTT
  • docker compose up -d shows containers as "Up" but lsof -i :1883 shows nothing on the host
  • Works fine on Linux but fails on macOS

Fixes

IssueFix
IPv6 binding Add socket_domain ipv4 to mosquitto.conf: 
listener 1883 0.0.0.0
socket_domain ipv4
allow_anonymous true
Apple Silicon (ARM64) Add platform: linux/amd64 to the Mosquitto service in docker-compose.yml
Docker Desktop networking stale Restart Docker Desktop entirely (not just the containers)
Port conflict Check with lsof -i :1883 and kill any existing process, or change the port mapping

Note: The provided mosquitto.conf already includes listener 1883 0.0.0.0 which handles most cases. If you encounter issues on macOS, the IPv4 socket domain fix is usually sufficient.
